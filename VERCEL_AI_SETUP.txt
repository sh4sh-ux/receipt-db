영수증DB Vercel AI 분석 서버 연결 방법

왜 바꾸나요?
- Cloudflare Worker에서 OpenAI API를 부르면 지역 제한 오류가 날 수 있습니다.
- 지금 나온 "OpenAI API 지역 제한" 오류는 앱 문제가 아니라 서버 위치 문제입니다.
- 그래서 분석 서버만 Vercel로 옮기면 더 안정적으로 테스트할 수 있습니다.

준비된 폴더
- vercel-ai-server

1. Vercel에 올리기
- https://vercel.com 에 로그인합니다.
- Add New... > Project 를 누릅니다.
- GitHub와 연결되어 있다면 새 저장소를 만들거나, 이 vercel-ai-server 폴더 내용을 올린 저장소를 선택합니다.
- Framework Preset은 Other로 두면 됩니다.
- Deploy를 누릅니다.

2. OpenAI API 키 넣기
- Vercel 프로젝트 화면에서 Settings > Environment Variables 로 갑니다.
- Add New를 누르고 아래처럼 넣습니다.

Name:
OPENAI_API_KEY

Value:
본인의 OpenAI API 키

- 키는 저에게 보내지 마세요.
- 값은 보통 sk-... 또는 sk-proj-... 형태입니다.
- 저장 후 다시 Deploy 해야 적용됩니다.

선택 값
OPENAI_MODEL = gpt-4.1-mini

3. 서버 주소 확인
- 배포가 끝나면 Vercel 주소가 생깁니다.
- 예: https://receipt-db-ai-server.vercel.app
- 뒤에 /api/analyze-receipt 를 붙여서 엽니다.

예:
https://receipt-db-ai-server.vercel.app/api/analyze-receipt

정상 화면:
{"ok":true,"name":"receipt-ai-vercel-server","model":"gpt-4.1-mini","ready":true}

4. 영수증DB 앱에 연결
- 영수증DB > 설정 > AI 자동 분석 서버
- 서버 주소에 Vercel 주소를 넣습니다.

예:
https://receipt-db-ai-server.vercel.app/api/analyze-receipt

- AI 연결 테스트
- AI 설정 저장
- 영수증 추가에서 사진을 선택해 테스트합니다.

중요
- OpenAI API 키는 GitHub나 index.html에 넣지 않습니다.
- Vercel Environment Variables에만 넣습니다.
- AI 분석값은 초안입니다. 저장 전 원본과 숫자를 직접 확인합니다.
