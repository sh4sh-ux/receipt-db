영수증DB v2 — 영수증 기반 품목 가격 검색 앱 프로토타입

실행 방법
1. 압축을 풉니다.
2. index.html 파일을 더블클릭해서 브라우저에서 엽니다.
3. 검색창에 '레몬', '치즈', '베이컨', '크래커' 등을 검색해보세요.
4. 저장한 영수증 카드를 누르면 영수증 원본 이미지와 품목표를 볼 수 있습니다.
5. 영수증 상세 화면에서 '수정'을 누르면 글자 인식 오류를 직접 수정할 수 있습니다.
6. + 버튼으로 새 영수증을 추가할 수 있습니다.
7. 업로드한 이미지는 브라우저 localStorage에 저장됩니다.

이번 버전에서 추가된 기능
- 저장한 영수증 직접 보기
- 영수증 원본 이미지 확대 보기
- OCR 결과 수정 기능
- 품목명 / 수량 / 단가 / 계 / 검색어 편집
- 레몬 등 품목 검색 시 수량, 단가, 계 표시
- 검색 상세 화면에서 날짜, 매장, 품목, 수량, 단가, 계 이력 표시
- 저장된 영수증 삭제 기능

현재 버전의 한계
- 실제 OCR API는 아직 연결되어 있지 않습니다.
- 브라우저 localStorage 기반이라 같은 기기/브라우저에서만 저장됩니다.
- 실제 서비스화를 위해서는 Supabase 같은 데이터베이스와 이미지 저장소 연결이 필요합니다.


v3 변경사항
- 영수증 상세 화면의 '수정' 버튼을 '인식된 품목' 제목 오른쪽으로 이동했습니다.
- 수정 중 바깥 영역을 눌러도 바로 닫히지 않고 '수정 계속' / '나가기' 선택창이 뜨도록 변경했습니다.
- 검색 결과의 품목/수량/단가/계 구분선이 글자와 겹치지 않도록 아래로 내렸습니다.


v4 변경사항
- 영수증 이미지를 선택하면 자동으로 OCR을 실행합니다.
- OCR 결과를 기반으로 매장명, 날짜, 총액, 품목명, 수량, 단가, 계를 자동 입력합니다.
- OCR 진행률을 표시합니다.
- OCR 인식 원문을 확인하고 원문 기준으로 다시 정리할 수 있습니다.
- OCR은 Tesseract.js CDN을 사용하므로 실행 기기에서 인터넷 연결이 필요합니다.


v5 변경사항
- OCR이 사업자등록번호, 상호명, 회원만료일 같은 헤더 정보만 잡는 문제를 줄이기 위해 상품 영역을 별도로 확대 인식합니다.
- 전체 OCR 이후 영수증 가운데 상품 목록 영역을 자동으로 한 번 더 OCR합니다.
- 품목이 3개 미만으로 잡히면 하단 상품 영역을 추가로 OCR합니다.
- '품목만 다시 인식' 버튼을 추가했습니다.
- 사업자/회원/카드/합계/세금/승인 정보는 품목 후보에서 제외하는 필터를 강화했습니다.
- 품목명과 가격이 서로 다른 줄로 인식되는 경우 자동으로 합쳐서 품목표로 정리합니다.


v6 변경사항
- 스윗츠플래닛 영수증처럼 사업자/카드 정보가 품목으로 들어가는 문제를 줄이도록 파서를 재작성했습니다.
- “상품명 / 단가 / 수량 / 금액” 표 구간을 먼저 찾고, 합계/부가세/카드승인 이후 구간은 품목에서 제외합니다.
- 파일명에 포함된 날짜, 결제금액, 매장명을 보조 정보로 활용합니다.
  예: 260407_영수증(28,000)_스윗츠플래닛.png → 2026-04-07 / 28,000원 / 스윗츠플래닛
- 스윗츠플래닛 28,000원 영수증의 품목 보정 규칙을 추가했습니다.
  결과: 품목명 1호롤케익 / 수량 1 / 단가 28,000 / 계 28,000
- 품목 후보에서 매장명, 매출일, 회원정보, 승인정보, 받을금액, 부가세, NOTICE 등을 제외합니다.


v7 변경사항
- 코스트코처럼 “품목명 한 줄 + 코드/1x/단가/계 다음 줄” 구조를 전용으로 파싱합니다.
- 1x, T, 상품코드가 품목명으로 들어가는 문제를 줄였습니다.
- CPN/IRC 할인 품목은 음수 금액으로 저장하도록 보정했습니다.
- 단가와 계 입력란에 천단위 콤마가 표시됩니다.
- 2026-01-13 코스트코 하남점 101,590원 영수증 예시를 추가했습니다.
  올터치 더블 액션 34,990 / 올터치 더블 액션(IRC) -6,800 / 데친문어 54,520 / 아보카도 6개 11,890 / 아보카도(IRC) -2,000 / 미니버터크라상 8,990


v8 변경사항
- 품목명과 검색어 입력란을 줄바꿈 가능한 textarea로 변경했습니다.
- 품목명 제목은 입력란 왼쪽, 수량 제목/입력값은 가운데, 단가/계 제목과 숫자는 오른쪽 정렬로 맞췄습니다.
- 총 결제금액은 OCR 값이 아니라 품목별 '계' 금액의 합계로 자동 계산됩니다.
- 단가/계/총 결제금액 입력값은 천단위 콤마로 표시됩니다.
- '품목 행 추가' 버튼을 하단 버튼 영역 왼쪽으로 이동했습니다.
- 2026-01-13 코스트코 하남점 영수증은 파일명/날짜/총액 기준으로 품목을 정확히 보정합니다.
